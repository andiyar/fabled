# Fabled decisions ledger

Append-only. Format per `~/Developer/fable-kit/decisions-ledger-template.md`: date · decision · why · revisit-when.

- **2026-07-08 · Pure Swift, no Node sidecar.** The Agent SDK just wraps the same CLI; a sidecar adds a runtime and an IPC hop to outsource a codec we can own. Tolerant decoding + fixtures make drift manageable. *Revisit if:* Anthropic breaks the stream-json protocol incompatibly more than ~once a quarter.
- **2026-07-08 · Depend on hidden flag `--permission-prompt-tool stdio`.** Only mechanism that turns auto-deny into interactive `can_use_tool` control requests; it is the SDK's own path, verified working on CLI 2.1.202. *Revisit if:* a CLI update removes/renames it (live tests will catch this immediately).
- **2026-07-08 · Opus subagents implement; coordinator (Fable while credits last, any model after) designs, dispatches, reviews.** Credit economics; the fable-kit method makes it work. Full handoff pack in `docs/superpowers/COORDINATION.md` + plan briefs.
- **2026-07-08 · ClaudeKit is zero-dependency.** SQLite via system C library, markdown via AttributedString, no GRDB/swift-markdown-ui. Keeps the engine auditable and the build trivial. SwiftTerm accepted later, app target only. *Revisit if:* AttributedString markdown proves unusable for code blocks (ledger the swap).
- **2026-07-08 · XcodeGen for the app target (Plan 3).** Agents can't drive Xcode GUI; `project.yml` is reviewable text; `.xcodeproj` becomes generated output. *Revisit if:* XcodeGen goes unmaintained.
- **2026-07-08 · Aesthetic: native macOS structure + Claude warmth in conversation** (serif voice, clay send button). Chosen by Ben over pure-native and dense-power-tool options.
- **2026-07-08 · Name: Fabled.** Chosen by Ben.
- **2026-07-08 · TranscriptEntry reshaped vs the Plan 2 brief.** Corpus census found zero `summary` lines (titles live in `custom-title`/`ai-title`), real prompts inside on-disk `user` lines, and `result` lines that are subagent caches, not turn results. Enum gains `userPrompt`/`title`/`sessionMeta` cases + a `LineContext` struct; `.summary` kept as legacy. *Revisit if:* the CLI changes the on-disk transcript format (the Task 11 drift census will show it).
- **2026-07-08 · Search schema: single FTS5 table, rowid = (file_id << 32) | line_no.** Per-file re-index becomes an indexed rowid range delete — no triggers, no external-content table. `files` carries a `title` column so search hits never re-open session files. Internal, not public API. *Revisit if:* the warm-reindex gate (<1 s) fails at real scale.
- **2026-07-08 · Index scope: human-relevant, main-chain text only.** Non-sidechain prompts (compaction summaries included), non-sidechain assistant text, titles, summaries. Sidechain/subagent traffic, `isMeta` lines, and tool results excluded. Subagent transcript files (`<session>/subagents/`) out of scope for Plan 2 entirely. *Revisit if:* users need to search subagent output.
- **2026-07-08 · Watcher: directory kqueue + 2 s mtime poll, throttled rescan.** Directory kqueue cannot see appends to existing files (POSIX), and appends are the main live signal; per-file kqueue would need 680+ fds. Poll stats depth-2 entries only. Throttle (not restartable debounce) so steady signals cannot starve the rescan. FSEvents stays rejected per the brief. *Revisit if:* poll cost shows up in profiles or fd budgets change.
- **2026-07-08 · JSONValue(parsing:) via JSONSerialization for bulk decode paths.** Several times faster than JSONDecoder for multi-MB transcript files; equivalence with the Plan 1 path is guard-tested over every fixture line. Live-session decoding keeps the original JSONDecoder path. *Revisit if:* the equivalence test ever fails on new fixtures.
- **2026-07-08 · Transcript perf gate amended 500 ms → 1 s.** Measured 634 ms, dead stable across runs, to fully decode the corpus-largest session (52.5 MB, 16,211 entries) after eliminating per-line copies (JSONLines slices). Cost is JSONSerialization-parse + bridge bound (~39 µs/line, ~83 MB/s); the 500 ms budget implicitly assumed a ~25 MB max file and the real max is 2×. Beating it needs a different decode architecture (custom parser / lazy raw payloads) — not warranted before Plan 3 UX proves the need. 1 s still trips on a JSONDecoder fallback, a double-parse, or quadratic behavior. *Revisit if:* Plan 3 transcript-open feels slow, or max file size keeps growing.
- **2026-07-09 · `updatedInput` is mandatory in allow responses.** Probed live: the CLI Zod-validates the allow payload and omitting `updatedInput` denies the tool (fixtures/2026-07-09-perm-allow-noinput.jsonl). `PermissionDecision.allow` reshaped to `(updatedInput:, updatedPermissions:)`; encoding echoes the request's input when nil. *Revisit if:* the CLI relaxes the schema (fixture re-record will show it).
- **2026-07-09 · Model picker is catalog-driven.** The initialize control_response carries a full `models` array (value, resolvedModel, displayName, effort caps); the picker renders it + a free-text custom ID. Replaces the brief's static alias list. *Revisit if:* the catalog disappears from the handshake.

- **2026-07-09 · `--resume` continues the SAME session ID — no fork-on-resume (probed, CLI 2.1.205).** Dead or live, `--resume <id>` reuses the id and appends to the same transcript file; only `--fork-session` mints a new identity. Two processes on one id interleave appends without corruption (line-atomic), so the one-process invariant guards *semantic* history cleanliness, not data integrity. The gate's "fork-like new session ID" observation was a UI-level artifact (Fabled adds a second live row next to the history row), which is exactly what feature 16 should fix: Continue = merge into the same identity; Fork = labelled branch. Fixtures: 2026-07-09-resume-collision-{A,B}.jsonl. *Revisit if:* a CLI update changes resume identity semantics (re-run fixtures/probe_collide.py). follow the user's `claude` symlink; detect drift from the install path at spawn.** The 2.1.204→2.1.205 auto-update retargeted `~/.local/bin/claude` mid-day and live Fabled sessions followed it (Claude Desktop was unaffected because it pins its own copy). Desktop-style pinning was considered and rejected: the installer garbage-collects old `versions/<v>` binaries (only 3 survive today), so a pinned path eventually dangles — a worse failure than drift — and a pinned Fabled silently diverges from the CLI Ben uses in the terminal (sessions, settings, features). Mitigations instead: (a) tolerant decoding stays the wire insurance, (b) `SessionConfiguration.resolveClaudeVersion()` reads the version out of the resolved symlink path at spawn and ChatSession warns immediately — `system init` can no longer be the trigger, 2.1.205 defers it until the first user turn, (c) fixtures re-record in minutes via `fixtures/probe_*.py`. *Revisit if:* a CLI update breaks the protocol outright (not just adds shapes) — then vendor a known-good binary per Fabled release.
